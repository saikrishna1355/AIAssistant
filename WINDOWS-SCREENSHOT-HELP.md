# Windows Screenshot Troubleshooting Guide\n\nIf you're seeing \"Windows fallback screenshot failed. Check screen capture permissions\", here's how to fix it:\n\n## Quick Solutions\n\n### 1. Run as Administrator (Easiest)\n1. Right-click on the Interview Copilot AI application\n2. Select \"Run as administrator\"\n3. Try taking a screenshot again\n\n### 2. Grant Screen Recording Permission\n1. Open **Windows Settings** (Windows key + I)\n2. Go to **Privacy & security**\n3. Click **Screen recording** in the left panel\n4. Toggle **\"Let apps access screen recording\"** to **ON**\n5. Find **Interview Copilot AI** in the list and toggle it **ON**\n6. Restart the application\n\n### 3. Fix PowerShell Execution Policy\n1. Open **PowerShell as Administrator**:\n   - Press Windows key + X\n   - Click \"Windows PowerShell (Admin)\" or \"Terminal (Admin)\"\n2. Run this command:\n   ```powershell\n   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser -Force\n   ```\n3. Restart the application\n\n### 4. Alternative: Use the Permission Checker\n1. In the application folder, double-click `check-windows-permissions.bat`\n2. Follow the on-screen instructions\n3. The script will help identify and fix permission issues\n\n## Advanced Solutions\n\n### If Screenshot Still Fails:\n\n1. **Disable Windows Defender Real-time Protection** (temporarily):\n   - Open Windows Security\n   - Go to Virus & threat protection\n   - Manage settings under Real-time protection\n   - Turn off Real-time protection temporarily\n   - Try screenshot again\n   - Re-enable protection afterward\n\n2. **Install Additional Screenshot Tools**:\n   - Download [nircmd](https://www.nirsoft.net/utils/nircmd.html)\n   - Extract to Windows folder or add to PATH\n   - This provides an additional screenshot method\n\n3. **Check Windows Version Compatibility**:\n   - The app works best on Windows 10 version 1903 or later\n   - Windows 11 has the best compatibility\n   - Older Windows versions may have limited screenshot API support\n\n## Automated Diagnostic\n\nThe application can automatically diagnose permission issues:\n\n1. When screenshot fails, click \"Yes\" to check permissions\n2. The app will show you exactly what's wrong\n3. Follow the specific suggestions provided\n4. Try the automatic fix if available\n\n## Common Error Messages\n\n- **\"PowerShell execution failed\"** → Run as Administrator or fix execution policy\n- **\"Screen capture permissions denied\"** → Enable screen recording in Windows Settings\n- **\"Access is denied\"** → Run as Administrator\n- **\"Permission check failed\"** → Windows security software is blocking access\n\n## Still Having Issues?\n\n1. **Restart Windows** after making permission changes\n2. **Update Windows** to the latest version\n3. **Check antivirus software** - some antivirus programs block screen capture\n4. **Try the web version** at http://localhost:3003 (limited functionality)\n\n## Technical Details\n\nThe screenshot feature uses multiple methods:\n1. Primary: `screenshot-desktop` Node.js library\n2. Fallback 1: PowerShell with System.Drawing\n3. Fallback 2: PowerShell with Windows.Graphics.Capture API\n4. Fallback 3: nircmd utility (if installed)\n\nIf all methods fail, it's typically a permission issue that can be resolved with the steps above.
